# Draft: Scribe Rebuild from Scratch

## Requirements (confirmed by user)

### KEEP IDENTICAL:
1. **UI Look & Feel**: All fonts, logos, icons, colors, animations must be preserved exactly
2. **BLE Microphone Workflow**: Proprietary SLink protocol - 8-step init, custom GATT services, Opus 16kHz mono
3. **Dual Recording**: Both built-in mic and BLE mic recording capability
4. **Flexible ML Pipeline**: Easy model replacement, transcript + summary + mind map generation
5. **Overview Screen**: All recordings list view
6. **Speaker Renaming**: Post-diarization rename for context

### CHANGE:
1. **Code Quality**: Increase cleanliness - eliminate 133 print statements, split god classes, proper error handling
2. **ML Pipeline Optimization**: Add VAD (speech detection before pipeline), language recognizer (auto-select ideal model)
3. **Unified Audio Format**: Same format for internal + BLE mic, small but sufficient quality for ASR
4. **Clean File Structure**: Eliminate Scribe/Scribe/Scribe nesting, flat structure for agentic coding

## Research Findings

### Current Structure Problems:
- **Triple nesting**: `Scribe/Scribe/Scribe/Sources/` - actual source 3 levels deep
- **23 Swift files** in 5 directories: Audio(6), Bluetooth(4), ML(2), Models(1), UI(8)
- **15 SPM dependencies**: FluidAudio, LLM.swift, swift-opus, swift-huggingface, swift-transformers, etc.
- **1 test file** only (BluetoothDeviceTests.swift with 33 tests)
- **No Info.plist** or entitlements in repo (auto-generated by Xcode)

### BLE Microphone (CRITICAL - Proprietary):
- **SLink Protocol**: Custom reverse-engineered, 8-step init sequence
  - Header: `0x80 0x08`, Commands: `0x02XX`, Checksum: CRC-16 variant with XOR `0x5F00`
  - Steps: Handshake(0x0202) → Serial("129950") → DeviceInfo(0x0201) → Configure(0x0204) → Status(0x0205) → 0x0218 → 0x020A → 0x0217
- **GATT Services**: Primary `E49A3001-F69A-11E8-8EB2-F2801F1B9FD1`
  - Audio Stream: `E49A3003-F69A-11E8-8EB2-F2801F1B9FD1`
  - Command Write: `F0F1`, File Transfer: `F0F2/F0F3/F0F4`, Battery: `2A19`
- **Audio Format**: Opus 16kHz mono → Float32 PCM via swift-opus
- **Known Devices**: LA518, LA519, L027, L813-L817, MAR-2518
- **Files**: DeviceConnectionManager.swift (676 lines), SLinkProtocol.swift (472 lines), BluetoothDevice.swift, AudioStreamReceiver.swift
- **Known Issues**: Hardware button detection TODO, checksum validation disabled, serial hardcoded

### ML Pipeline (4 stages, sequential):
1. **Diarization**: TitaNet-Small via FluidAudio, FP16/ANE, ~0.8GB RAM
2. **Audio Load**: AudioConverter (CAF only - BUG: fails on .m4a), 16kHz Float32
3. **Transcription**: NVIDIA Parakeet-TDT-0.6B-v3 via FluidAudio, ~1.3GB RAM
4. **LLM Summary**: Llama-3.2-3B-Instruct-Q4_K_M via llama.cpp, ~2.1GB RAM

**Missing**: VAD stage, language detection, model swapping config
**Bugs**: AudioConverter only handles CAF, 48kHz→16kHz mismatch for internal mic

### UI System:
- **Theme.swift**: 5 colors (scribeRed, obsidian, cardBackgrounds, accentGray), scribeCardStyle modifier
- **Dark mode forced** via .preferredColorScheme(.dark)
- **SF Symbols only** - no custom fonts
- **6 main views**: RecordingListView, RecordingDetailView, RecordingCardView, RecordButtonView, AgentGeneratingView, DeviceSettingsView
- **Animations**: Pulsing record button (1.5s repeat), mesh gradient (4s repeat), circle pulse, progress spring

### Code Quality Issues:
- **133 print statements** across 10 files (62 in DeviceConnectionManager alone)
- **God class**: DeviceConnectionManager at 676 lines handles connection + SLink + audio + heartbeat
- **Empty catch block** in AudioRecorder.swift line 139
- **Duplicated code**: formatDuration() in 2 files, RSSI threshold logic repeated
- **Magic numbers**: UUIDs, serial "129950", timeouts, sample rates all hardcoded
- **Tight coupling**: Direct references between services, no protocol-based DI
- **Missing docs**: Public APIs lack documentation comments

## Technical Decisions (Confirmed)

1. **Unified Audio Format**: **Opus for both** internal and BLE microphones
2. **VAD**: **FluidAudio's built-in VAD** — avoids adding new dependencies
3. **Language Detection**: **Undecided** — need to detect Swiss German specifically to route to fine-tuned model. Options: NLLanguageRecognizer (built-in), fasttext-langdetect, or custom approach

6. **Target iOS**: **iOS 18+** — user devices all run iOS 26.4 (note: likely means iOS 18.x)

## Technical Decisions (Confirmed)

### ASR Strategy — Swiss German First
- **Primary ASR**: Swiss German Whisper CoreML model — `jlnslv/whisper-large-v3-turbo-swiss-german-coreml` from HuggingFace
- **Fallback**: If snippet is NOT Swiss German → switch to backup model (Parakeet or general Whisper)
- **Classifier**: Use Whisper's built-in language detection confidence as YES/NO signal for Swiss German
- **Config-driven**: Model paths and names in a config file, easy to swap without code changes

### Model Serving
- **HuggingFace** is the primary source for CoreML models
- **FluidAudio** offers a great selection of models — use if available there
- If models are NOT in FluidAudio, serve CoreML models directly from HuggingFace

### Xcode Project Structure
